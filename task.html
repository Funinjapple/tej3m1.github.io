TEJ3M1-01 Unit 1 Assignment 3: Separating Structure and Presentation with XHTML and CSS

Due Date:  Thursday, September 22, 2016   Marks:  APPL:  /30

The aim of this assignment is to learn how structure and presentation of web content can be separated using XHTML and CSS.

Task 1: Learn XHTML and CSS

Learn to write eXtensible HyperText Markup Language (XHTML) and Cascading Style Sheets (CSS) by following the tutorials at http://www.w3schools.com/css/.

Task 2: Create a Web Page [ /20]

Create a web page on some topic that you know something about. Also create an external style sheet to give web browsers hints about how you prefer the pages to be rendered. The files should fulfill the following criteria:

  • Use strict XHTML.
  • Comment your code appropriately.
  • Use at least one image and one table.
  • Use meta tags to describe the content of your pages.
  • Define at least three different CSS classes in your style sheet and demonstrate their use in the XHTML pages.
  • Read up on CSS pseudo-classes and define two classes of hyperlinks: one where the links turn green when the mouse hovers over them and one where the links turn red.
  • Testing with Multiple HTTP Clients. Beginning with this assignment, you are expected to test your work with at least two different HTTP clients (Web browsers). Suggestions for a second HTTP client (I assume you already are using Microsoft Internet Explorer) are: Mozilla Firefox , Opera , Netscape , Amaya You can get to Firefox through: q: utilities

Make sure you separate structure and presentation as much as possible. Use indentation to make sure your code is readable and make sure you have all the closing tags in place.

Task 3: Validate Your Pages [ /10]

Use the World Wide Web Consortium's (W3C) XHTML validation service at http://validator.w3.org/ to check that your page conforms to the XHTML and CSS standards. You can either upload the page to the W3C or put it online and type the URL into the W3C's validator form.

You will run your program for the teacher on the due date. Be prepared to verbally explain to the teacher any aspect of your programming code.

Marks will be allocated for clear separation of structure and presentation and for generally good programming style, i.e., adequate use of tags, indentation, etc.
